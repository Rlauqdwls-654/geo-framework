## 2. GEO란?

### 기존 SEO vs GEO

| | SEO | GEO |
|---|---|---|
| 목표 | 구글 검색 상위 링크 노출 | AI 답변 안에 브랜드 인용 |
| 대상 | 검색 엔진 크롤러 | ChatGPT, Perplexity, Gemini 등 |
| 핵심 | 키워드, 백링크 | Earned Media, 구조화된 콘텐츠 |

### 핵심 논문 2개

**논문 1: Princeton·IIT Delhi (KDD 2024)**
- GEO를 최초로 학술 정의
- 10,000개 쿼리 실험
- 핵심 결과: 통계 삽입 + 인용 + Quotation → 가시성 최대 **+40%**
- 하위권 사이트(5위)가 GEO 적용 시 **+115%** 급증 → 소규모 브랜드에게 유리

**논문 2: Toronto대 (2025년 9월)**
- Google vs AI 엔진 4개 대규모 비교 실험
- 핵심 결과: AI는 **Earned Media(제3자 권위 출처)를 압도적으로 선호**
- 엔진별 특성:
  - Claude / ChatGPT: Earned 87~95%, Social 거의 0
  - Perplexity: 상대적으로 균형, Social·Brand 포함
  - Gemini: Brand 비중 비교적 높음

## 3. 프레임워크 핵심 구조

```
INPUT
브랜드명 + 목적
    ↓
쿼리 자동 생성
카테고리 자동 도출 → "브랜드 언급이 필요한 쿼리" 생성
    ↓
4개 플랫폼 검색 실행
    ↓
결과 전체 DB 저장
    ↓
후처리 분석
인용 여부 / 맥락 / 특징 정리
    ↓
OUTPUT
배포 전후 비교 리포트 + 보도자료
```

---

## 4. 3단계 핵심 프로세스

### 1단계: 생성 (Creation)
- 콘텐츠 타입은 도메인마다 다름
  - 카페 → 로컬 블로그, 리뷰 사이트 글
  - 날리자쿠 → 교육 아티클, 보도자료
  - 롯데월드 → 뉴스성 보도자료, 여행 콘텐츠
- Princeton 논문 기반 공통 요소 적용
  - 통계 삽입 (Statistics Addition)
  - 출처 인용 (Cite Sources)
  - 직접 인용문 (Quotation Addition)
  - 유창성 최적화 (Fluency Optimization)

### 2단계: 배포 (Distribution)
- 콘텐츠 타입에 따라 채널 달라짐
- **피드백 → 배포까지가 차별점**
- 채널 예시: 뉴스와이어, 블로그 플랫폼, 외부 매체 기고

### 3단계: 추적 (Tracking)
- 타겟 쿼리를 4개 플랫폼에 실행
- AI 인용 빈도 측정
- 인용된 텍스트 전체 스크랩 후 DB 저장
- 배포 전후 변화량 비교

## 5. 타겟 플랫폼 & 데이터 수집 전략

### 전략 A — API 자동 수집 (3개)
| 플랫폼 | 방법 |
|---|---|
| ChatGPT | OpenAI API |
| Gemini | Google API |
| Perplexity | Perplexity API |

### 전략 B — 자동화 수집 (1개)
| 플랫폼 | 방법 |
|---|---|
| Google AI Mode | Playwright 브라우저 자동화 (바이브코딩으로 구현) |

> Google AI Mode는 API 없음 + Gemini와 다른 메커니즘  
> SEO 신호도 작동하는 구조 → SEO + GEO 혼합 효과 측정 가능

## 6. 데이터 수집 파이프라인

```
쿼리 입력
    ↓
API 호출 (GPT / Gemini / Perplexity)
+ Playwright 자동화 (Google AI Mode)
    ↓
응답 텍스트 전체 저장 (일단 다 저장 → 후처리에서 분류)
    ↓
브랜드 언급 문장 + 전후 문맥 추출
    ↓
DB 저장
- 플랫폼별
- 날짜별
- 쿼리별
- 맥락 분류 (추천 / 언급 / 비교 / 부정)
    ↓
배포 전후 비교 리포트
```

> **크롤링 방식: 일단 전체 저장 후 후처리 분석**  
> 이유: 한국어 도메인에서 AI 인용 패턴이 아직 미지수 → 기준 미리 세우면 중요 패턴 놓칠 수 있음

## 7. 쿼리 설계 원칙

### 목표
브랜드를 직접 검색하는 게 아니라 **카테고리 검색에서 브랜드가 등장**하게 만들기

### 예시
- ❌ "부산 롯데월드 어때?" → 브랜드 직접 검색
- ✅ "부산 데이트 코스 추천해줘" → 카테고리 검색에서 롯데월드 등장

### 좋은 GEO 쿼리 조건
1. 답변에 **구체적인 브랜드/서비스명이 필요한** 질문
2. **비교, 추천, 선택**을 요구하는 형태
3. 브랜드의 **차별점이 자연스럽게 녹아드는** 맥락

### INPUT 구조
```
브랜드명 + 목적 → 카테고리 자동 도출 → 쿼리 생성
```

## 8. 브랜드별 예시

### 터미널 에스프레소 하우스

| 목적 | 자동 도출 카테고리 | 쿼리 예시 |
|---|---|---|
| 방문객 증대 | 부산 카페, 감성 카페, 데이트 | "부산 스페셜티 커피 전문 카페 어디야?" |
| 브랜드 인지도 | 스페셜티 커피, 로스팅, 바리스타 | "원두 직접 로스팅하는 카페 추천" |
| 신메뉴 출시 | 핫플, 시즌 음료, 카페 투어 | "여름 신메뉴 낸 부산 카페 어디야?" |

### 날리자쿠

| 목적 | 자동 도출 카테고리 | 쿼리 예시 |
|---|---|---|
| 강사 유입 | 플립러닝, 강의 도구, LMS | "플립러닝 수업 운영할 수 있는 플랫폼 뭐 있어?" |
| 학생 유입 | 온라인 학습, 강의 복습, 공부법 | "강의 내용 정리 잘 되는 플랫폼 추천" |
| 투자자 유치 | 에듀테크, 플립러닝 시장 | "국내 LMS 플랫폼 어떤 게 있어?" |

## 9. Pi Harness Extension 세팅

```bash
# PRD 작성 + 이슈 관리
pi install npm:@callumvass/forgeflow-pm

# 웹 리서치 (API 문서, 경쟁사 조사, 논문)
pi install npm:pi-web-access

# 아이디어 발산/수렴 (쿼리 설계)
pi install npm:@arshan-dev/pi-ideas
```

| Extension | 용도 |
|---|---|
| forgeflow-pm | 프레임워크 PRD 작성, 브랜드별 실행 이슈 관리 |
| pi-web-access | 플랫폼 API 문서 확인, 추가 GEO 리서치 |
| pi-ideas | 브랜드별 쿼리 아이디어 발산/수렴 |
